What we did

* Remplacement de la lib bitronix => narayana-jta (=arjuna)
* Adaptation de la conf et des paramètres
* Extraction / wrapping de l'utilisation du transaction manager dans une classe Bonita pour faciliter le changement d'implem par la suite

Spécificités / Difficultés:

* bitronix fournit une datasource / un pool de connexion * narayana, non. Du coup, utilisation de Apache Commons DBCP2 qui supporte les pools / datasources XA

Achievements

* Tous les tests moteur H2 tournent * Tests sur toutes les Base de Données en cours: http://ymci.rd.lan/blue/organizations/jenkins/bonitasoft%2Fbonita-engine-sp/detail/PR-1199/1/pipeline

Reste à faire

* Faire plus de tests * Checker tous les paramètres possibles de configuration

Bienfaits

* meilleur compréhension de comment s'articule ces briques (JTA transaction manager, UserTransaction, datasource, pool, configuration dans les AppServer) * A l'oeil humain, la rapidité d'exécution semble bonne * Va permettre d'embarquer le Transaction Manager dans le moteur * => Car il est compatible avec tous les SGBD, contrairement à Bitronix (incompatible avec SQL Server) * => => Donc va permettre de supprimer les bundles fournis par Bonita !
